Изучить принципы построения Hardware Abstraction Layer и научиться писать переносимые драйверы периферийных устройств.

### Теоретическая часть

HAL (Hardware Abstraction Layer) — это уровень абстракции аппаратного обеспечения, который обеспечивает независимость прикладного ПО от конкретного микроконтроллера. HAL определяет абстрактные интерфейсы для работы с периферией.

```
┌─────────────────────────────────────┐
│        Прикладное ПО                │
├─────────────────────────────────────┤
│        HAL интерфейсы (IUART, ISPI) │
├──────────────┬──────────────────────┤
│ STM32Driver  │   OtherDriver        │
│ (конкретная  │   (другой МК)        │
│ реализация)  │                      │
└──────────────┴──────────────────────┘
```

## Лабораторная работа 2: CAN шина

### Цель работы

Освоить работу с CAN интерфейсом, изучить протокол CANopen для организации бортовой сети.

### Теоретическая часть

CAN (Controller Area Network) — надёжный последовательный интерфейс с дифференциальной сигнализацией. Используется как основная шина обмена данными между модулями спутника.

**Формат CAN кадра:**
```
┌─────┬───────┬─────┬──────┬──────┬──────┬──────┬──────┬─────┐
│ SOF │  ID   │ RTR │ IDE  │ DLC  │ DATA │ CRC  │ ACK  │ EOF │
│ 1 b │ 11 b  │ 1 b │ 1 b  │ 4 b  │0-8 B │ 15 b │ 2 b  │ 7 b │
└─────┴───────┴─────┴──────┴──────┴──────┴──────┴──────┴─────┘
```

**CANopen Object Dictionary:**
| Index | Объект |
|-------|--------|
| 0x1000 | Device Type |
| 0x1001 | Error Register |
| 0x1018 | Identity Object |
| 0x2000+ | Manufacturer Specific |
